01 -- Why This Exists

The Gap: No Official Anaplan MCP Server

Anaplan has no official MCP server. Nobody does.
Every interaction requires manual work or custom code -- from scratch, every time.
What doesn't exist (until now)
No official Anaplan MCP server -- from Anaplan or any vendor
No standard way for AI assistants to query or act on Anaplan data
No shared layer for auth, pagination, retry across integrations
LLMs can reason over planning data but have zero native access
What we built
An open-source MCP server that wraps the full Anaplan Integration API v2 in 70 structured tools -- so any MCP-compatible AI can talk to Anaplan out of the box. No glue code. No manual steps. No waiting.

2. Handle authPOST to auth endpoint, store token, track expiry, refresh before 35-min window closes

One natural-language prompt now does all of this automatically.

02 -- Foundation

What is MCP?

Model Context Protocol -- the open standard by Anthropic that makes this possible

MCP gives AI assistants a structured, tool-calling interface to external systems -- similar to how REST standardised web APIs. The AI describes what it wants in natural language; MCP routes it to the right tool with validated parameters.
AI Assistant
Claude · Cursor · Copilot
→
Anaplan MCP
JSON-RPC · stdio · local
→
Anaplan API
api.anaplan.com
Plug-and-play

Any MCP-compatible host connects automatically. No custom integration per AI tool.

Self-describing tools

Each tool has a name, description, and validated schema. The AI decides when to call it.

Local and secure

Runs as a local process. Credentials never leave your machine or reach any cloud relay.

The Anaplan-specific gap

MCP servers exist for GitHub, Slack, databases, filesystems, Jira, and dozens of other enterprise tools. Anaplan has none -- official or community. This is the first.

What this unlocks for Anaplan
Ask Claude to pull model data -- it calls the right API, handles auth and pagination
Say "run the January import" -- it triggers, polls, and reports back with row counts
Ask "list all models in this workspace" -- instant, no UI navigation needed
Write data values using plain dimension names, not 32-char hex IDs
Works today with Claude Desktop, Claude Code, and Cursor -- more AI hosts every month

07 -- How It Works

Architecture

Three layers under src/ -- each with a single responsibility

Tools Layer
exploration.ts
bulk.ts
transactional.ts
70 MCP tools registered on McpServer. zod-validated input params. formatTable() returns markdown tables as separate content blocks. All list tools include limit and search filtering. Built-in orchestration guide resource teaches AI the correct tool sequences.
Naming: show_ for exploration, run_/set_/create_/delete_ for bulk, direct verbs for transactional. Polymorphic tools route via actionType param.
API Layer
client.ts
+ 16 wrappers
AnaplanClient handles all HTTP to api.anaplan.com/2/0/. Retry on 429 (Retry-After) and 5xx -- 3 attempts, exponential backoff 1s to 4s. getAll() auto-paginates. getRaw() for CSV chunks.
16 domains: Workspaces, Models, Modules, Lists, Imports, Exports, Processes, Files, Actions, Transactional, Dimensions, ModelMgmt, Calendar, Versions, Users, LargeReads.
Auth Layer
basic.ts
certificate.ts
oauth.ts
AuthManager selects provider by env var priority: certificate > OAuth > basic. Caches token, schedules auto-refresh 5 min before the 35-min Anaplan expiry. Zero manual management.
OAuth: device grant (interactive) or authorization code grant (non-interactive). Certificate: PEM cert + private key via env vars.
NameResolver spans all layers -- accepts "Q4 Forecast" or raw 32-char hex IDs. Sequential resolution: workspace then model then module. Case-insensitive, 5-min in-memory cache. Orchestration guide resource provides multi-step workflow patterns to the AI.

08 -- Technical Depth

Under the Hood

The engineering details that make this production-ready, not a prototype

Retry & Resilience

Exponential backoff on 5xx -- 3 attempts
Delays: 1s, 2s, 4s. 429s honour the Retry-After header exactly. No silent failures -- errors surface with full context.
// client.ts
if (res.status === 429) {
  const wait = retryAfter ?? 1000 * 2**attempt;
  await sleep(wait);
}
Auto-pagination with getAll()
Fetches every page using meta.paging. Guards against Anaplan returning undefined (not []) for empty arrays. Supports fallback key arrays for inconsistent response shapes.
// keeps fetching until no next page
while (meta?.paging?.nextPage) {
  offset += limit;
  results.push(...await fetch(offset));
}

Auth & Token Lifecycle

Token auto-refresh -- never expired mid-request
35-min Anaplan token lifetime. AuthManager schedules refresh 5 min early. Long-running operations never hit a 401 mid-way through.
// manager.ts
refreshAt = expiresAt - 5 * 60 * 1000;
setTimeout(refresh, refreshAt - Date.now());
Name to ID resolution with caching
Every tool accepts plain names. Sequential resolution: workspace then model then module. Results cached 5 min to avoid redundant API calls.
const wId = await resolver.resolveWorkspace(workspaceId);
const mId = await resolver.resolveModel(wId, modelId);
// mId is now the 32-char hex ID

09 -- Governance

Security & Safety

What stays local, what never leaves, and what the AI can and cannot do

Credentials never leave your machine

Server runs as a local subprocess over stdio. Your Anaplan credentials are used directly in API calls -- never sent to Claude's servers or any cloud relay.

Explicit intent required for writes

37 exploration tools are fully read-only. Write tools and job triggers only fire when the AI explicitly calls them -- with visible parameters you can review and approve.

Your existing credentials and permissions

No new service accounts, no elevated access. The server operates within whatever Anaplan permissions your account already has.

Open source and fully auditable

The entire codebase is open source on GitHub. Every API call appears as a tool call in the AI chat before it runs.
